> 版本: 1.0
> 日期: 2025-12-30
> 参考: Suno, Linear, Vercel, ElevenLabs, Stripe

---

## 1. 设计理念

### 1.1 核心定位

**不是又一个 AI 工具，而是你的音乐创作伙伴。**

区别于 Suno/Udio 的"文字生成音乐"定位，我们强调：
- **人声输入** → 更有温度的创作方式
- **哼唱即创作** → 降低门槛，人人都是创作者
- **AI 编曲** → 保留你的旋律灵感，AI 负责专业制作

### 1.2 视觉风格

| 维度 | 方向 | 参考 |
|-----|------|------|
| 主题 | **深色主题** | Suno, Linear |
| 氛围 | 沉浸、专业、有温度 | - |
| 动效 | 流畅、克制、有意义 | Vercel, Linear |
| 排版 | 大留白、强对比、层次分明 | Stripe |

### 1.3 差异化要点

❌ 避免：
- 蓝紫渐变（AI 工具标配）
- 科技感过重的几何图形
- 冷冰冰的机器人形象
- 过度炫技的 3D 效果

✅ 追求：
- 温暖的深色调（深紫 + 琥珀点缀）
- 音乐相关的视觉隐喻（声波、音符、节奏）
- 人文气息（真实用户、情感连接）
- 精致的微交互

### 3.1 Navigation Bar

**交互**：
- 滚动 > 50px 后显示背景
- Mobile 汉堡菜单，侧滑抽屉

### 3.2 Hero Section ⭐ 核心

**视觉概念**：声波从中心向外扩散，象征音乐的诞生

**布局**：
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│                    ♪  ♫  ♪                                  │
│              (浮动音符粒子动画)                               │
│                                                             │
│         ════════════════════════════                        │
│              (动态声波可视化)                                 │
│         ════════════════════════════                        │

│                                                             │
│              哼 一 段 ， 成 一 曲                             │
│                                                             │
│         你的旋律灵感，AI 帮你变成完整音乐                      │
│                                                             │
│                                                             │
│              [ 🎤 开始录音 ]    [ ▶ 听听效果 ]               │
│                                                             │
│                                                             │
│              已有 12,580 首作品诞生                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**设计规格**：

| 元素 | 规格 |
|-----|------|
| 容器高度 | 100vh (首屏全占) |
| 背景 | 深色渐变 `#0a0a0f` → `#1a1025` |
| 主标题 | 72px / 4.5rem，字重 700，渐变文字 |
| 副标题 | 20px / 1.25rem，字重 400，`neutral-400` |
| 主 CTA | 56px 高，渐变背景，发光效果 |
| 次 CTA | 56px 高，透明边框，hover 填充 |

**动效设计**：

1. **声波可视化** (Hero 背景)
   - Canvas 绘制，响应音频或模拟数据
   - 颜色：主色渐变 `primary-500` → `primary-700`
   - 透明度：0.3，不抢主体
   - 动画：持续波动，周期 3s

2. **浮动音符**
   - 5-8 个音符图标随机分布
   - 缓慢上下浮动，周期 4-6s（错开）
   - 透明度 0.2-0.4
   - 鼠标移动时轻微视差

3. **文字入场**
   - 主标题：fade-up，延迟 0.2s
   - 副标题：fade-up，延迟 0.4s
   - 按钮：fade-up，延迟 0.6s
   - 统计数字：fade-in，延迟 0.8s

4. **CTA 按钮**
   - Hover：scale(1.02) + 发光增强
   - 录音按钮：脉冲动画暗示可点击

## 5. 响应式断点

| 断点 | 宽度 | 布局调整 |
|-----|------|---------|
| Mobile | < 640px | 单列，缩小字号，简化动效 |
| Tablet | 640-1024px | 两列网格，保留核心动效 |
| Desktop | > 1024px | 完整布局，全部动效 |

### 5.1 Mobile 适配要点

- Hero 标题：48px → 36px
- 流程卡片：横向滚动
- 功能网格：单列堆叠
- 评价卡片：全宽，左右滑动
- 导航：汉堡菜单

## 6. 性能优化

### 6.1 加载策略

1. **首屏优先**
   - Hero 区域 CSS 内联
   - 关键字体 preload
   - 背景动画延迟加载

2. **懒加载**
   - 非首屏图片
   - 视频/音频资源
   - 第三方脚本

3. **代码分割**
   - 按区块动态导入
   - 动画库按需加载

### 6.2 动效性能

- 使用 `transform` 和 `opacity` 动画
- 避免 `layout` 触发属性
- Canvas 动画使用 `requestAnimationFrame`
- 低端设备降级处理
